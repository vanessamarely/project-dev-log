Actúa como un Senior Frontend Developer y mentor especializado en UX/UI moderno.

Necesito que transformes mi proyecto frontend "Dev-Log" en una versión más avanzada y visualmente atractiva, como una mini aplicación moderna tipo dashboard de productividad y aprendizaje.

Objetivo del proyecto:
Dev-Log es una bitácora de aprendizaje donde el usuario puede crear memo fichas o cards para registrar conceptos, aprendizajes, errores solucionados, ideas y avances técnicos.

La aplicación debe sentirse como una mini app moderna de escritorio:
- visualmente limpia,
- oscura,
- elegante,
- interactiva,
- y preparada para seguir evolucionando durante un curso de Git y GitHub.

Tecnologías:
- HTML5 semántico
- CSS3 moderno:
  - variables CSS
  - gradientes
  - glassmorphism
  - animaciones
  - hover effects
  - responsive design
- Vanilla JavaScript ES6+
  - usar const/let
  - usar arrow functions
  - usar template literals
  - usar funciones pequeñas y reutilizables
  - mantener código limpio y modular
  - no usar sintaxis legacy ES5
- Sin frameworks
- Sin librerías externas

Estructura obligatoria:

dev-log/
├── index.html
├── style.css
├── script.js

Diseño visual esperado:
- Fondo oscuro azul noche (#0f172a o similar).
- Hero superior moderno con:
  - título grande “Dev-Log”
  - gradiente morado/celeste
  - subtítulo motivacional
  - badge decorativo superior
- Layout moderno tipo dashboard.
- Panel superior ancho para crear entradas.
- Diseño responsive.
- Glassmorphism suave en paneles y cards:
  - background translúcido
  - blur
  - sombras suaves
  - bordes sutiles
- Cards modernas con:
  - fecha
  - categoría
  - acciones
  - hover elevation
  - borde glow suave
- Botón principal con gradiente y animación hover.
- Botón flotante circular decorativo abajo a la derecha.
- Espaciado amplio y diseño limpio.
- Tipografía moderna y jerarquía visual clara.

Funcionalidades requeridas:
- Crear nuevas entradas dinámicamente.
- Mostrar fecha automática.
- Mostrar categoría seleccionada.
- Categorías:
  - Aprendizaje
  - Git
  - JavaScript
  - UI/UX
  - Error solucionado
- Mostrar badge visual para cada categoría.
- Insertar nuevas cards al inicio.
- Actualizar contador de entradas.
- Limpiar formulario después de guardar.
- Animación popIn al crear nuevas cards.
- Remover clase de animación al terminar.
- Botón para eliminar entradas.
- Confirmación antes de eliminar.
- Mostrar mensaje vacío si no existen entradas.
- Mantener responsive design en móviles.

Requisitos visuales adicionales:
- Grid responsive de cards.
- Sombras modernas suaves.
- Hover elegante en cards y botones.
- Glow sutil en elementos interactivos.
- Bordes redondeados modernos.
- Scroll elegante si hay muchas entradas.
- Microinteracciones suaves.

Requisitos técnicos:
- No usar innerHTML para insertar texto del usuario.
- Usar textContent cuando sea posible.
- Mantener el código organizado.
- Separar claramente responsabilidades entre HTML, CSS y JS.
- Agregar comentarios breves solo en partes importantes.
- No usar frameworks ni librerías externas.

Entrega requerida:
1) Comandos de terminal para crear el proyecto.
2) Contenido completo de:
   - index.html
   - style.css
   - script.js
3) Explicación breve de las funcionalidades implementadas.
4) Indicaciones para abrir el proyecto en el navegador.

Resultado esperado:
Una mini aplicación moderna y visualmente atractiva que se sienta como una verdadera app frontend profesional, pero construida únicamente con HTML, CSS y JavaScript vanilla.
